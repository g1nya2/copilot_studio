# Copilot Studio 핸즈온: 나만의 무드 타입 테스트봇 만들기

## 0. 오늘 만들 것

오늘은 Microsoft Copilot Studio만 사용해서 **무드 타입 테스트봇**을 만듭니다.

이 에이전트는 사용자의 답변을 바탕으로 다음 결과를 만들어줍니다.

* 나의 무드 타입
* 성향 키워드
* 추천 활동
* 오늘의 미션
* 안전 점검 결과
* 친구에게 공유할 짧은 카드

최종 구조는 아래와 같습니다.

```text
메인 에이전트
 ↓
Knowledge: mood_types.txt
 ↓
Connected agents 4개
- 타입판정가
- 활동추천가
- 결과평가가
- 안전검토가
 ↓
Skills 6개
- mood-type-test
- p1-draft-card
- p2-review-result
- p3-improve-card
- p4-safety-check
- p5-share-card
 ↓
최종 결과 출력
```

## 1. 준비물

수업을 시작하기 전에 아래가 필요합니다.

```text
□ Copilot Studio 접속 계정
□ Microsoft 계정 로그인
□ mood_types.txt 파일
□ 인터넷 연결
```

모델은 화면 오른쪽 위에서 **GPT-5 Reasoning**으로 설정합니다.

## 2. 새 에이전트 만들기

Copilot Studio에 접속한 뒤 아래 순서대로 진행합니다.

```text
Agents
→ New agent
→ Skip to configure
```

에이전트 이름은 아래처럼 입력합니다.

```text
무드 타입 테스트
```

메인 에이전트 화면의 큰 지침 칸에 아래 내용을 붙여넣습니다.

```text
너는 중학생을 위한 재미있는 무드 타입 테스트봇이야.

목표:
학생의 기분, 주말 활동, 친구들이 보는 모습, 좋아하는 분위기를 바탕으로 재미용 무드 타입 카드를 만들어준다.

반드시 다음 순서로 작업한다.

1. 사용자가 "무드 타입 테스트", "내 타입 알려줘", "취향 테스트", "MBTI 같은 거 해줘", "오늘의 무드 알려줘"라고 말하면 mood-type-test skill을 시작한다.

2. 최종 결과를 바로 만들지 말고, 반드시 아래 5가지를 차례대로 질문한다.
- 실명이 아닌 별명
- 오늘 기분
- 주말에 끌리는 활동
- 친구들이 나를 표현하는 말
- 좋아하는 분위기나 색감

3. 사용자의 답변을 모두 받은 뒤 Knowledge에 있는 mood_types.txt, 즉 "무드 타입 데이터"를 참고한다.

4. Connected agent "타입판정가"를 사용해 학생에게 가장 가까운 무드 타입을 고른다.

5. Connected agent "활동추천가"를 사용해 선택된 무드 타입에 맞는 추천 활동, 오늘의 미션, 친구와 할 만한 활동을 만든다.

6. p1-draft-card skill을 사용해 1차 무드 타입 카드를 만든다.

7. Connected agent "결과평가가"와 p2-review-result skill을 사용해 1차 결과가 재미있는지, 안전한지, 중학생에게 적절한지 평가한다.

8. p3-improve-card skill을 사용해 평가 결과를 반영한 최종 무드 타입 카드로 개선한다.

9. Connected agent "안전검토가"와 p4-safety-check skill을 사용해 외모 평가, 몸매 평가, 인기 비교, 개인정보 요구, 상처 줄 표현이 없는지 점검한다.

10. p5-share-card skill을 사용해 친구에게 공유하기 좋은 짧은 카드 문구를 만든다.

11. 마지막에는 최종 무드 타입 카드, 안전 점검 결과, 공유용 카드를 모두 보여준다.

중요 규칙:
- 실명, 전화번호, 주소, 학교명 같은 개인정보를 묻지 않는다.
- 별명만 사용한다.
- 외모, 몸매, 성적 매력 평가는 하지 않는다.
- 인기 순위, 친구 비교, 따돌림을 부추기는 표현은 하지 않는다.
- 결과는 반드시 재미용 테스트라고 표현한다.
- 성격을 단정하지 않는다.
- "너는 무조건 이런 사람이야"가 아니라 "이런 무드에 가까워 보여요"라고 말한다.
- 결과는 밝고 긍정적으로 작성한다.
- 중학생이 이해하기 쉬운 말투를 사용한다.
- 최종 결과는 너무 짧게 쓰지 말고 충분히 자세하게 작성한다.
- 가능한 경우 연결된 에이전트와 skill을 순서대로 활용한 것처럼 각 단계의 결과를 반영한다.

최종 출력 형식:

[최종 무드 타입 카드]

별명:
무드 타입:
한 줄 소개:
성향 키워드 3개:
타입 설명:
나의 매력 포인트:
잘 맞는 친구 타입:
추천 활동 5개:
오늘의 미션 3개:
친구에게 공유할 한 줄:
30초 발표 대본:
해시태그:

[안전 점검 결과]

검토 결과:
수정 필요한 표현:
최종 통과 여부:

[친구에게 공유하는 짧은 카드]

3줄 요약:
친구에게 보내는 한 줄:
다음 테스트 질문:
```

## 3. Knowledge 파일 만들기

메모장을 열고 아래 내용을 그대로 붙여넣습니다.

파일 이름은 반드시 아래처럼 저장합니다.

```text
mood_types
```

파일 내용은 아래를 사용합니다.

```text
무드 타입 데이터

1. 몽글감성 힐링러
키워드: 조용함, 감성, 음악, 기록, 공감
설명: 분위기를 잘 느끼고 친구를 편하게 해주는 타입
잘 맞는 친구 타입: 텐션업 분위기메이커
추천 활동: 감성 플레이리스트 만들기, 폰 배경 꾸미기, 짧은 일기 쓰기
오늘의 미션: 친구에게 고마운 점 하나 말하기

2. 텐션업 분위기메이커
키워드: 활발함, 웃김, 리액션, 친구, 장난
설명: 주변 분위기를 밝게 만들고 친구들을 웃게 해주는 타입
잘 맞는 친구 타입: 몽글감성 힐링러
추천 활동: 친구들과 사진 포즈 정하기, 웃긴 순간 기록하기
오늘의 미션: 친구 한 명에게 재밌는 칭찬 해주기

3. 트렌드 탐험가
키워드: 유행, 쇼츠, 패션, 아이돌, 밈
설명: 새로운 유행을 빠르게 발견하고 친구들에게 알려주는 타입
잘 맞는 친구 타입: 차분한 분석가
추천 활동: 요즘 유행하는 밈 정리하기, 좋아하는 콘텐츠 추천 리스트 만들기
오늘의 미션: 친구에게 새로 알게 된 유행 하나 알려주기

4. 카리스마 리더
키워드: 결정력, 발표, 계획, 책임감, 정리
설명: 친구들을 이끌고 상황을 정리하는 힘이 있는 타입
잘 맞는 친구 타입: 아이디어 뱅크
추천 활동: 학교 행사 아이디어 내기, 모둠 역할 정리하기
오늘의 미션: 작은 계획 하나 세워서 실천하기

5. 조용한 천재
키워드: 혼자 집중, 독서, 그림, 관찰, 생각
설명: 말은 많지 않아도 깊게 생각하고 자기만의 세계가 있는 타입
잘 맞는 친구 타입: 텐션업 분위기메이커
추천 활동: 나만의 노트 꾸미기, 캐릭터 설정 만들기
오늘의 미션: 오늘 떠오른 생각 한 줄 적기

6. 아이디어 뱅크
키워드: 상상, 만들기, 콘텐츠, 기획, 창의력
설명: 새로운 아이디어를 잘 떠올리고 뭔가를 만들어보고 싶어하는 타입
잘 맞는 친구 타입: 카리스마 리더
추천 활동: 새로운 앱 아이디어 3개 적기, 상상한 캐릭터 설명하기
오늘의 미션: 친구에게 재미있는 아이디어 하나 말해보기

7. 공감 상담러
키워드: 친구 고민, 대화, 배려, 듣기, 따뜻함
설명: 친구의 이야기를 잘 들어주고 마음을 이해하려는 타입
잘 맞는 친구 타입: 트렌드 탐험가
추천 활동: 친구 고민에 답장 문구 만들기, 위로 문장 모으기
오늘의 미션: 친구 한 명에게 응원 메시지 보내기

8. 완벽주의 계획러
키워드: 계획, 정리, 체크리스트, 꼼꼼함, 준비
설명: 차근차근 준비하고 실수를 줄이려는 타입
잘 맞는 친구 타입: 아이디어 뱅크
추천 활동: 내일 할 일 체크리스트 만들기, 책상 정리 루틴 만들기
오늘의 미션: 오늘 한 일 3개 체크하기

9. 시크한 관찰자
키워드: 차분함, 관찰, 센스, 조용함, 분석
설명: 겉으로는 조용하지만 상황을 잘 보고 센스 있게 반응하는 타입
잘 맞는 친구 타입: 텐션업 분위기메이커
추천 활동: 친구들의 말버릇 관찰하기, 좋아하는 장면 캡처해 감상 적기
오늘의 미션: 오늘 발견한 센스 있는 순간 하나 기록하기

10. 반짝반짝 꾸미기 장인
키워드: 꾸미기, 색감, 다꾸, 폰꾸, 감각
설명: 색감과 분위기를 잘 보고 자기 취향을 표현하는 타입
잘 맞는 친구 타입: 조용한 천재
추천 활동: 폰 배경 컨셉 만들기, 다이어리 한 페이지 꾸미기
오늘의 미션: 오늘의 색깔 하나 정하고 이유 적기

11. 콘텐츠 몰입러
키워드: 웹툰, 드라마, 아이돌, 영상, 스토리
설명: 좋아하는 콘텐츠에 깊게 빠지고 이야기의 매력을 잘 느끼는 타입
잘 맞는 친구 타입: 트렌드 탐험가
추천 활동: 좋아하는 캐릭터 매력 분석하기, 추천 콘텐츠 리스트 만들기
오늘의 미션: 친구에게 콘텐츠 하나 추천하기

12. 편안한 집순이 무드
키워드: 집, 휴식, 간식, 음악, 편안함
설명: 편안한 공간에서 에너지를 충전하고 자기만의 시간을 좋아하는 타입
잘 맞는 친구 타입: 공감 상담러
추천 활동: 나만의 힐링 루틴 만들기, 쉬는 날 플레이리스트 만들기
오늘의 미션: 오늘 나를 편하게 해준 것 하나 적기

13. 핑크빛 하이틴 주인공
키워드: 사진, 친구, 하이틴, 밝음, 꾸미기
설명: 일상도 영화 장면처럼 즐기고 싶은 타입
잘 맞는 친구 타입: 반짝반짝 꾸미기 장인
추천 활동: 친구와 콘셉트 사진 찍기, 오늘의 OOTD 설명하기
오늘의 미션: 오늘 하루를 영화 제목처럼 지어보기
```

## 4. Knowledge 업로드하기

```text
Knowledge
```

그 옆의 `+` 버튼을 누릅니다.

```text
Knowledge +

```

```text
mood_types.txt
```

## 5. Skills 만들기

오른쪽 패널에서 아래 항목을 찾습니다.

```text
Skills
```

`Skills +` 버튼을 누르고 아래를 선택합니다.

```text
빈칸에서 만들기
```

주의: `Tools +`가 아닙니다.
이 UI에서는 프롬프트 역할을 하는 작업들을 **Skills**로 만듭니다.

총 6개의 Skill을 만듭니다.

```text
mood-type-test
p1-draft-card
p2-review-result
p3-improve-card
p4-safety-check
p5-share-card
```

Skill 이름은 영어로 작성합니다.

---

## 5-1. Skill 1: mood-type-test

### 이름

```text
mood-type-test
```

### 설명

```text
사용자가 무드 타입 테스트, 취향 테스트, MBTI 같은 거, 오늘의 무드, 내 타입 알려줘 같은 말을 하면 사용하는 기술입니다. 학생에게 별명, 오늘 기분, 주말 활동, 친구들이 보는 모습, 좋아하는 분위기를 차례대로 질문하고, Knowledge의 무드 타입 데이터를 참고해 최종 무드 타입 카드를 만들어줍니다.
```

### 지침

```text
When this skill is activated:

1. 학생에게 실명 대신 별명만 물어본다.
2. 오늘 기분을 물어본다.
3. 주말에 끌리는 활동을 물어본다.
4. 친구들이 자신을 어떻게 표현하는지 물어본다.
5. 좋아하는 분위기나 색감을 물어본다.
6. Knowledge에 있는 "무드 타입 데이터"를 참고한다.
7. 타입판정가 connected agent를 사용해 가장 가까운 무드 타입을 고른다.
8. 활동추천가 connected agent를 사용해 추천 활동과 오늘의 미션을 만든다.
9. p1-draft-card skill을 사용해 1차 카드를 만든다.
10. p2-review-result skill을 사용해 결과를 평가한다.
11. p3-improve-card skill을 사용해 최종 카드를 개선한다.
12. p4-safety-check skill을 사용해 위험한 표현을 점검한다.
13. p5-share-card skill을 사용해 친구에게 공유할 짧은 카드를 만든다.
14. 최종 카드와 공유용 카드를 사용자에게 보여준다.

## Guidelines

- 실명, 전화번호, 주소, 학교명 같은 개인정보를 묻지 않는다.
- 별명만 사용한다.
- 외모, 몸매, 성적 매력 평가는 하지 않는다.
- 인기 순위, 친구 비교, 따돌림을 부추기는 표현은 하지 않는다.
- 결과는 재미용 테스트라고 표현한다.
- 성격을 단정하지 않는다.
- "너는 무조건 이런 사람이야"가 아니라 "이런 무드에 가까워 보여요"라고 말한다.
- 결과는 밝고 긍정적으로 작성한다.
- 중학생이 이해하기 쉬운 말투를 사용한다.

## Examples

User request:
```

최종 결과에 아래 3개 섹션이 나오면 성공입니다.

```text
[최종 무드 타입 카드]

[안전 점검 결과]

[친구에게 공유하는 짧은 카드]
```

---

## 10. 게시하기

테스트가 성공하면 오른쪽 위 버튼을 누릅니다.

```text
게시하기
```

게시가 완료되면 다른 사용자도 에이전트를 사용할 수 있습니다.

## 12. 자주 발생하는 문제

### 문제 1. Skills 이름을 한국어로 만들 수 없어요.

Skill 이름은 영어로 작성합니다.

```text
mood-type-test
p1-draft-card
p2-review-result
p3-improve-card
p4-safety-check
p5-share-card
```

설명과 지침은 한국어로 작성해도 됩니다.

---

### 문제 2. Tools에서 프롬프트를 찾을 수 없어요.

이 실습에서는 `Tools +`를 사용하지 않습니다.

오른쪽 패널에서 아래를 사용합니다.

```text
Skills +
→ 빈칸에서 만들기
```

---

### 문제 3. 에이전트가 질문하지 않고 바로 결과를 만들어요.

메인 지침 맨 위에 아래 문장을 추가합니다.

```text
사용자가 테스트를 요청하면 절대 바로 결과를 만들지 말고, 반드시 별명, 오늘 기분, 주말 활동, 친구들이 보는 나, 좋아하는 분위기나 색감을 차례대로 질문한 뒤 결과를 만들어야 한다.
```

---

### 문제 4. Knowledge를 참고하지 않는 것 같아요.

메인 지침에 아래 문장을 추가합니다.

```text
최종 결과를 만들기 전에 반드시 Knowledge에 업로드된 mood_types.txt의 "무드 타입 데이터"를 참고해야 한다.
```

---

### 문제 5. Connected agent를 안 쓰는 것 같아요.

메인 지침에 아래 문장을 추가합니다.

```text
최종 결과를 만들기 전에 반드시 타입판정가, 활동추천가, 결과평가가, 안전검토가 connected agent의 역할을 순서대로 수행해야 한다.
```
